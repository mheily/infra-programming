path to the script

When deploying to the global namespace, use a site-specific prefix. Typically this is constructed from the company's name; e.g Acme Software would use the prefix “as”. A command named foo could have the path /usr/bin/as-foo.

It would be even better to have a separate path for site-specific scripts. Then users can add this to their PATH. Example: /acme/bin/foo
